Gene annotation of GPL10787 with R

GEO platform annotation without bioconductor package

Monday, 23 January 2023

Commomly, researchers have developed many R packages for gene annotation and host them in Bioconductor. For most data series from NCBI Gene Expression Omnibus (GEO), we can find annotation packages. Yet, we may encounter with some data series without such packages, such as GSE94522.

Previously, 4 ways to ðŸ‘‰Get GEO platform annotation file and import into R were introduced, which can be accessed by clicking the name underlined and in italic.

Here shares are the scripts for gene annotation of data series GSE94522 from GEO.

rm(list = ls())

# load R packages
# library(GEOquery)
# library(clusterProfiler)
# library(org.Mm.eg.db)

# load GEO platform data
gseID <- "GSE94522" # platformID <- "GPL10787"
load(paste0(gseID, ".GPL.Rdata"))

# check gpl with View() OR head()
# result of View() shown in Figure 1
View(gpl)
colnames(gpl)
# Result:
[1] "ID"           "SPOT_ID"        "CONTROL_TYPE"   "REFSEQ"             "GB_ACC"                "GENE"            "GENE_SYMBOL"        "GENE_NAME"           
[9] "UNIGENE_ID"   "ENSEMBL_ID"     "TIGR_ID"        "ACCESSION_STRING"   "CHROMOSOMAL_LOCATION"  "CYTOBAND"        "DESCRIPTION"        "GO_ID"               
[17] "SEQUENCE"

look through the result table and look up gene IDs. In Figure 1, we will find genes in Column 5-10 in different formats. In addition, we will see probes for CONTROLs both POSITIVE and NEGATIVE as labeled in Column 3 with column named with CONTROL_TYPE. Just delete them as below:

# Delete probes for controls
gpl <- gpl[gpl$CONTROL_TYPE == 'FALSE',]

For gene annotation, probe IDs and genes are sufficient. So we need to extract these columns and delete unwanted ones. Here, we extract columns named with ID (Column 1), GENE (Column 6) and GENE_SYMBOL (Column 7).

# Delete unwanted columns
gpl <- gpl[, c(1, 6:7)]
head(gpl)

save(ids, file = "ids.Rdata")
