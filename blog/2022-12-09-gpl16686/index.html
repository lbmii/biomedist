Gene annotation of GPL16686 with R

GEO platform annotation without bioconductor package

Friday, 09 December 2022

Commomly, researchers have developed many R packages for gene annotation and host them in Bioconductor. For most data series from NCBI Gene Expression Omnibus (GEO), we can find annotation packages. Yet, we may encounter with some data series without such packages, such as GSE179445.

Previously, 4 ways to 👉Get GEO platform annotation file and import into R were introduced, which can be accessed by clicking the name underlined and in italic.

Here shares are the scripts for gene annotation of data series GSE179445 from GEO.

rm(list = ls())

# load R packages
library(GEOquery)
library(clusterProfiler)
library(org.Hs.eg.db)

# load GEO platform data
gseID <- "GSE179445" # platformID <- "GPL16686"
load(paste0(gseID, ".GPL.Rdata"))

# check gpl with View() OR head()
# result of View() shown in Figure 1
View(gpl)

look through the result table and look up gene IDs. In Figure 1, we will find gene IDs in Column 6 named GB_ACC.

# Trandform types of gene IDs with bitr {clusterProfiler}
# type of GB_ACC genes is defined as "ACCNUM"
gene.Entrezid<- bitr(gpl[, 6],
                     fromType = "ACCNUM",
                     toType = c("SYMBOL","ENTREZID"),
                     OrgDb = org.Hs.eg.db)
# Result:
'select()' returned 1:1 mapping between keys and columns
Warning message:
In bitr(gpl[, 6], fromType = "ACCNUM", toType = c("SYMBOL", "ENTREZID"),  :
  3.63% of input gene IDs are fail to map...

head(gene.Entrezid) # OR dim(gene.Entrezid)
# Result:

  • Pay attention to the Warning message when running bitr().
  • 3.63% of input gene IDs are fail to map… means probes without annotation exist.
# delete probes without annotation
probes.sense <- gpl[gpl$GB_ACC %in% gene.Entrezid$ACCNUM,]
dim(probes.sense) # OR View(probes.sense) OR head(probes.sense)

# validate the column number of gene IDs
colnames(probes.sense)
# Result:
[1] "ID"           "RANGE_STRAND" "RANGE_START"  "RANGE_END"    "total_probes" "GB_ACC"       "SPOT_ID"      "RANGE_GB"

"ID" (probe ID) and "GB_ACC" (gene ID) are respectively located in Column 1 and Column 6.

# extract probes and gene IDs
probes.sense <- probes.sense[,c(1,6)]
colnames(probes.sense)
# Result:
[1] "ID"     "GB_ACC"

colnames(probes.sense) <- c("probe_id", "ACCNUM")

Now we have got genes as SYMBOLs, but they are in different tables with probes. So we need to merge them together.

# check the data type of probes and genes
class(probes.sense)
# Result:
[1] "data.frame"

class(gene.Entrezid)
# Result:
[1] "data.frame"

# merge two data frames of probes and genes
ids <- merge(probes.sense, gene.Entrezid, by = "ACCNUM")
head(ids)
# Result:
     ACCNUM probe_id SYMBOL ENTREZID

3 NM_000019 16730819  ACAT1       38

dim(ids)
# Result:
[1] 16936     4

Finish with saving ids as Rdata, which can be loaded with Load() quickly.

save(ids, file = "ids.Rdata")
